Pokémon Legends Arceus: Pokédex Cover

Overview

This main goal of this project was to learn how to implement an inventory/bestiary system. I picked the Pokédex from Pokémon Legends: Arceus because it was the most feature-complete, contemporary example at the time of the project. The secondary goal was to practice making a good User Interface. For me being a game designer means thinking of every facet of the game, from its mechanics and feedback to the way its presented to the player.

Project Goals

  • •Learn how to make an inventory/bestiary system
  • •Study UI design decisions from a AAA perspective
  • •Pratice improving workflow

Lessons From Project

Tool Creation: One of my personal challenges for this project was creating a tool that would make editing the Pokédex as easy as possible. This tool took values from a spreadsheet and converted them into Pokédex entries. By frontloading the work, I made it incredibly easy to make progress on the project from anywhere.

Inventory Management: Item Systems and Bestiaries serve similar functions: show the player information on objects with an emphasis on ease-of-use. This project helped me realize which elements of a list style system were most important and different ways to implement them.

Have Fun With It: This cover is one of my most feature complete because I found the fun in making the project itself. I may not choose what I work on, but I can make the experience easier if I go into it with the right mindset.

Design Process Summary

For this cover my approach can be summerized as follows:
    • Research and Questions
    • Planning
    • Implementation
    • Bonus Content

Research and Questions

The first step for this project was to research the system I was trying to cover, including its functions and inputs. What are the players’ motivations for interacting with this system? How do they navigate to that goal?

I believe that figuring out the intention of a system is just as important as learning how to make it. What is the purpose of the Pokédex? What are the main functions that players expect? What types of inputs were used? These questions help me fully understand the system I’m trying to recreate, instead of just copying it.



Planning

My next task is to plan out how this system works. Below is a list of the functions I believe are crucial to the presentation of the Pokédex and why:

Individual Entries: The main function of the Pokédex is to display information on whatever Pokémon is selected IF the player has already caught them. This means that each entry must have unique information stored in a way that doesn’t take up too much space.

Sprite Changer: As the player cycles through different Pokédex entries, the picture of the Pokémon displayed changes. This is one of the most important features for making sure the information in the Pokédex is conveyed correctly.

Easy UI: Finding information in a Pokédex needs to be fast and intuitive. Overall the system needs to be simple enough for a child to navigate it with ease. This means big pictures, cohesion across the entire application, and simple inputs.


Implementation

The original design for this involved creating prefabs for each Pokémon with every detail attached. For me that was too cumbersome and slow, so I wanted to find a better way. I researched different ways to have static data loaded in that could be easily edited. It was here I found the pipeline of taking a Google Sheet, downloading the data as a .csv (Comma Separated Value), convert that into a .json (JavaScript Object Notation), then have Unity read that file. After this I was able to work on the project everywhere, leading to a much more robust set of content.



Bonus Content

Editing ended up being so easy that I had a chance to add other features I wanted. Here’s a few and a brief description of each:

Music: Opening the Pokédex plays the National Park theme from Pokémon HeartGold/SoulSilver. I wanted this to be a relaxing experience, so the music fit perfectly.

Shiny Hunting: Pokémon have a rare chance (about 1 in 4,096) to be a different color, complete with visual and sound effects. I added this to the Pokédex, so every Pokémon has both a regular and shiny variant. Each time you “catch” a Pokémon in this application, a number is rolled to determine if you encountered the shiny.

Togglable Entries: If you catch a shiny or alpha and display its information, the Pokédex will remember next time you look at that entry.
